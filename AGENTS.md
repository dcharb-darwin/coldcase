# AGENTS

> Master orchestration contract for Cold Case. Every agent that works in this repo must follow these rules.

> This file is the IDE-agnostic entry point. Claude Code, Cowork, Codex, Cursor, and Windsurf all respect `AGENTS.md` by convention.

## Mandatory init sequence

Before doing anything in this repo, read these in order:

1. **`docs/PRINCIPLES.md`** — design intent (P1–P6 + agentic)
2. **`docs/PATTERNS.md`** — architectural patterns (provider/adapter, admin pattern, JSON-mode, audit, env-driven ports)
3. **`docs/CONVENTIONS.md`** — coding conventions (naming, 500-line cap, commit trailers)
4. **`docs/AGENTIC.md`** — if touching AI assistant code
5. **`docs/PLAYBOOK.md`** — lifecycle procedures
6. **`agents/memory-bank/current-state.md`** — module status
7. **`SESSION_STATE.md`** — if continuing prior work

## Commit contract

Every feat/fix/chore commit:
- Trailer: `[trace: coldcase-<concern>]`
- Git identity: `dcharb-darwin <daniel.charboneau@darwingov.com>`
- Never `--no-verify`, never `--amend` pushed commits, never force-push main.

## Gating rules

- Architecture-relevant files changed → before `git push`, a tracked `.githooks/pre-push` reminds you to regenerate `docs/architecture.md`.
- Behavior changed → update `docs/comprehensive-prd.md` changelog before the session ends.
- Module moved → update `STRUCTURE.md` and `agents/memory-bank/module-registry.json`.

## Non-negotiables (from `docs/PLAYBOOK.md §10`)

- No `--no-verify` on commits.
- No `--amend` on pushed commits.
- No `reset --hard` on `main`.
- No secrets committed.
- No `any` in TypeScript.
- No hard-coded ports.
- No raw hex / rgb in component files.
- Commit trailers `[trace: <slug>]` on every feat/fix/chore.
