## What we learned

**Pitch in one line:** governance + audit layer wrapping the agency's GCC Copilot so detectives' AI use is SB-524-compliant.

**Critical constraints:**
- Documents stay in the agency's Azure/S3. Cold Case stores **metadata + lineage only**.
- Analysis runs in the agency's **GCC Copilot tenant**, not ours.
- Every prompt + response (including discarded drafts) is preserved.
- Approved AI artifacts must be stamped "generated with AI Copilot by <user>".
- Long retention: homicide = effectively forever; default 2y, configurable 7y / indefinite.

**Deliberately deferred (Dan flagged scope creep on the call):**
- Cross-document timeline / relationship graph.
- Audio / video ingestion.
- Cold Case running its own AI analysis.

## What's now built

### Frontend (`src/`)
- **API client** (`lib/api/coldcase.ts`) — typed bindings for every cases / conversations / reports / audit endpoint.
- **Routes** — `/cases` (list), `/cases/:id` (3-pane detail), `/audit` (timeline). Hash-routed. Nav rail + breadcrumbs updated.
- **`CaseListPage`** — table + "New case" modal. Homicide selection surfaces the §13663(b) indefinite-retention auto-suggest.
- **`CaseDetailPage`** — three panes: documents/media/reports sidebar (with inline registration forms), document viewer, chat panel.
- **`ChatPanel`** — conversation switcher, document/media context picker (`§13663(c)(2)` audited per send), assistant messages with a `📌 §13663(b) first draft (locked)` indicator and "Use as official report →" promote action.
- **`ReportDrawer`** — promote → edit → sign → export flow. §13663(a)(1) disclosure banner, first-AI-draft read-only panel, officer attestation block with name/badge inputs (sign required before export). Renders signature payload + content hash after signing.
- **`AuditPage`** — filterable event table.

### Backend (`server-py/`)
- **App manifest** (`server-py/auth/app_manifest.py`) — 16 permissions across Cases / Documents / AI / Reports / Audit / Admin; 5 seed roles (admin, detective, sergeant, auditor, viewer).
- **Domain models** (`server-py/models/`) — Case, Document, MediaInput, Conversation, Message, Report (with AIProgram + OfficerSignature embedded docs), Approval, AuditEvent. Indexes set per PRD §6.
- **Provider seams** (`server-py/providers/`) — `llm.py` (Ollama mock, **OpenAI provider** reading `OPENAI_API_KEY` + `OPENAI_MODEL`, GCC Copilot stub) and `document_storage.py` (local-fs mock + Azure Blob stub). The exact model id OpenAI returns (e.g. `gpt-4o-mini-2024-07-18`) is captured on every assistant Message and surfaces on the §13663(a)(1) disclosure footer.
- **Routers** (`server-py/routers/`) — `cases.py` (F1), `conversations.py` (F2), `reports.py` (F3), `audit.py` (F4). All wired in `server.py`.
- **Disclosure PDF export** (`server-py/services/report_export.py`) — verbatim §13663(a)(1) disclosure on every page, AI-programs identification, officer-attestation block with hashed e-signature, Appendix A renders the §13663(b) first AI draft labeled "not an officer statement".
- **Smoke test** (`/tmp/coldcase-smoke.py` + `/tmp/coldcase-smoke2.py`) — 17 steps, 9 explicitly statute-driven, all green against a freshly-booted stack on `:7787`. PDF verified via `pypdf` extraction on both rendered pages.
