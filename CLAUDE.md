## Architecture map

The live architecture description is auto-maintained at `docs/architecture.md` (regenerated by the user-global `architecture-diff-gate` skill). The top-level shape is:

```
server-py/              ← FastAPI backend (MongoDB via MongoEngine)
├── server.py           ← App factory — see docs/PATTERNS.md §2 admin pattern wiring
├── core/               ← config, database, dev_auth_bypass
├── auth/app_manifest.py← THIS APP's permissions + seed roles (hand-authored)
├── launchpad_admin/    ← 8-file RBAC package, verbatim from launchpad-admin-pattern
├── routers/            ← one file per domain entity
├── models/             ← MongoEngine documents
├── services/           ← business logic + ollama_client + admin_assistant + audit
└── providers/          ← mock/real adapters (employee, email, calendar, training, evaluation, photos)

src/                    ← React frontend (Vite + TypeScript)
├── main.tsx            ← QueryClient + UserContextProvider + configureAdminApi
├── App.tsx             ← AppRouter
├── shell/              ← P3 navigation shell (AppShell, routes, breadcrumbs)
├── launchpad-admin/    ← 14-file RBAC UI, verbatim
├── components/         ← Darwin primitives + ImpersonationBanner
├── lib/api/            ← axios client + interceptors
├── features/           ← feature-scoped pages/components/hooks
└── index.css           ← Darwin design tokens (P5)
```
